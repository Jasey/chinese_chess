# 中国象棋 UI 应用 — 需求文档

## 概述

Python 桌面象棋应用，支持人机对战和 AI 自对弈，AI 通过 LLM SDK 驱动。

---

## 功能需求

### 1. 游戏模式

| 模式 | 说明 |
|------|------|
| 人机对战 | 玩家执红或黑，AI 执另一方 |
| AI 自对弈 | 红黑双方均由 AI 控制，自动轮流走棋 |

### 2. 棋局规则

#### 棋盘
- 10 行 × 9 列交叉点
- row 0 = 黑方底线，row 9 = 红方底线
- 楚河汉界在 row 4/5 之间

#### 初始布局

| 位置 | 棋子 | 颜色 |
|------|------|------|
| row 0, col 0-8 | 車馬象士將士象馬車 | 黑 |
| row 2, col 1/7 | 砲 | 黑 |
| row 3, col 0/2/4/6/8 | 卒 | 黑 |
| row 6, col 0/2/4/6/8 | 兵 | 红 |
| row 7, col 1/7 | 炮 | 红 |
| row 9, col 0-8 | 車馬相仕帥仕相馬車 | 红 |

#### 走法规则

| 棋子 | 规则 |
|------|------|
| 车 | 直线任意格，不可翻山越岭 |
| 马 | 日字形，蹩腿不可走 |
| 象/相 | 田字形，不可过河，塞象眼不可走 |
| 士/仕 | 斜线一格，不出九宫 |
| 将/帅 | 直线一格，不出九宫，不可与对方将帅对脸（飞将） |
| 炮/砲 | 平移不吃子，隔一子方可吃子 |
| 兵/卒 | 未过河只能向前；过河后可横走（仍不可后退） |

#### 终局判定
- **将死**：被将军且无合法应将走法 → 对方胜
- **困毙**：未被将军但无合法走法 → 平局

### 3. 人机对战交互

- 点击己方棋子：高亮选中，显示合法落点（绿色小圆点）
- 点击合法落点：执行走法
- 点击己方其他棋子：切换选中
- 点击无效位置：取消选中
- AI 思考时禁止点击，显示"AI 思考中…"
- 支持悔棋：同时撤销玩家和 AI 各一步

### 4. AI 玩家

- 调用 LLM SDK（`llm_client.py`）获取走法
- Prompt 包含棋盘文本 + 当前方所有合法走法列表
- 解析返回格式：`from_row,from_col to_row,to_col`
- 最多重试 3 次；全部失败则随机选一个合法走法

### 5. 界面

- tkinter Canvas 绘制棋盘（楚河汉界、九宫斜线、兵炮位标记）
- 棋子：圆形底色 + 汉字，红方红底白字，黑方黑底白字
- 状态栏：当前回合、将军提示、胜负结果
- 按钮：重新开始、悔棋（仅人机模式）

---

## 非功能需求

- AI 思考在独立线程执行，不阻塞 UI
- AI 自对弈每步间隔 ≥ 600ms，避免 CPU 占用过高
- LLM 请求超时 120s
- 代理地址可在启动时配置

---

|------|--------|------|

| 我执棋色 | 红方 | 红方（先手）/ 黑方；AI 自对弈时不生效 |

