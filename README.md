# 中国象棋 Web 应用 — 当前需求说明

## 概述

本项目当前主交付物为本地 Web 版中国象棋应用，提供三种核心场景：

- 人机对战
- AI 自对弈
- 棋谱分析 / 逐手复盘

应用通过本地 HTTP 服务提供前端页面，棋局规则由 Python 实现，AI 对弈与分析由 Pikafish 驱动，讲解由本地规则总结或 LLM 教练生成。

## 当前功能范围

### 1. 起局页与对局页分离

- 起局页负责模式、执棋颜色、模型、引擎路径、线程数、分析时长等配置
- 对局页负责棋盘、战报、教学讲解、最近几手、棋谱详情与归档查看
- 支持继续当前内容与重新开始

### 2. 模式

| 模式 | 说明 |
|------|------|
| 人机实战 | 玩家执红或黑，AI 执另一方，主界面默认弱化讲解，强调局中节奏 |
| 人机教学 | 每一步落下后强制等待分析完成，随后按当前回合自然继续 |
| AI 自对弈 | 双方均由 AI 控制，可暂停 / 继续 |
| AI 对弈教学 | AI 每步也先完成讲解，再自动进入下一手 |
| 棋谱分析 | 导入 ICCS 棋谱，后台逐手分析并生成讲解 |

### 3. 棋盘与规则

- 10 行 × 9 列中国象棋标准棋盘
- 完整支持车、马、象/相、士/仕、将/帅、炮/砲、兵/卒规则
- 支持将军、将死、困毙、飞将合法性过滤
- 支持 ICCS 走法与中文着法表示

### 4. 教学与复盘

- 每步可记录：
  - 中文着法
  - ICCS 着法
  - 是否吃子 / 将军 / 接近首选
  - 引擎多主线评分
  - LLM / 本地生成讲解
- 教学模式中，分析未完成时必须锁盘
- 右侧讲解看板支持展示当前步摘要、主线参考、三段式讲解
- “最近几手”可点击回看，并将该步内容重新绘制到讲解看板
- 棋谱详情支持查看完整逐手记录，并可送去棋谱分析

### 5. 声音系统

- 落子音效、环境配乐可独立开关
- 模式配乐采用分模式环境化编配：
  - 实战：更强推进感的钢琴 / 低音氛围
  - 教学：更克制的弦乐讲解感
  - 复盘：更空灵、丝弦化的回看氛围
- 棋子音效强调木案、石子与低频共振，覆盖：
  - 拿子
  - 落子
  - 吃子
  - 将军
  - 妙手提示
  - 胜负收尾

### 6. 归档

- 最近对局自动归档
- 归档保存模式、棋谱、逐手讲解、分析完成状态
- 棋谱分析支持复用已分析归档，避免重复计算

## 交互约束

### 人机实战

- 玩家只能在自己回合操作
- AI 思考或讲解生成时禁止落子
- 支持悔棋、认输、重新开始

### 教学模式

- 无“继续下一手”强制按钮
- 任意一步落下后，必须等待分析完成
- 分析完成后：
  - 若轮到玩家，由玩家自行决定何时落子
  - 若轮到 AI，则自动进入下一手

### AI 自对弈

- 支持暂停 / 继续
- 暂停时不应丢失当前棋局状态

### 棋谱分析

- 支持粘贴空格分隔的 ICCS 棋谱
- 分析过程可见进度
- 已分析归档再次打开时应直接复用结果

## 非功能要求

- 本地运行，不依赖外部在线页面
- AI 分析与讲解在后台线程执行，不阻塞页面响应
- 前端状态需能在刷新后恢复最近配置
- 模式切换时，界面与音乐过渡需要自然，不应出现硬切
- 讲解、战报、最近几手的信息层次要明确，避免互相抢注意力
